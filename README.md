Краткое описание игры
Реализован 3D раннер: игрок (персонаж с "CharacterController") автоматически бежит вперёд по бесконечной дорожке, уклоняется по 3 линиям (лево/центр/право), прыгает через препятствия, получает урон при столкновениях. При потере всего здоровья игра ставится на паузу и показывается экран поражения с кнопкой перезапуска.

Точки входа (главная сцена и ключевые скрипты)
- Главная сцена:сцена с индексом 0 в Build Settings (она же загружается при рестарте).
- Основная логика игры: GameManager управляет окончанием игры: показывает LosePanel и останавливает время.
- Игровой цикл игрока: PlayerController - движение вперед, смена линий, прыжок, здоровье/урон и вызов GameManager.EndGame() при смерти.
- Генерация окружения: TileGenerator — бесконечно добавляет сегменты дорожки и удаляет старые позади игрока.
- Генерация препятствий: ObstacleSpawner  — периодически спавнит препятствия (2 типа) на случайной линии впереди игрока и очищает объекты позади.
- Камера: FollowCamera — следование камеры за игроком по оси Z с постоянным смещением.
- UI: Score — вывод очков; LosePanel — кнопка перезапуска (загрузка сцены 0).
- Ввод: SwipeController — формирует события свайпов/тапа (мышь/тач) и поддерживает клавиши A/D/Space.

Алгоритм работы 
1. Старт сцены
   - TileGenerator создаёт стартовые сегменты дорожки.
   - ObstacleSpawner инициализирует позицию следующего спавна впереди игрока.
   - PlayerController выставляет здоровье, включает время и запускает корутину постепенного увеличения скорости.
   - FollowCamera запоминает смещение относительно игрока.

2. Основной цикл
   - PlayerController:
     - в FixedUpdate() постоянно двигает игрока вперёд, применяет гравитацию и прыжок;
     - в Update() обрабатывает смену линии и прыжок по данным из SwipeController.
   - TileGenerator по мере продвижения игрока создаёт новые сегменты дорожки и удаляет старые.
   - ObstacleSpawner с заданной вероятностью спавнит препятствия впереди игрока (случайная линия/дистанция/тип) и удаляет те, что остались позади.
   - Score обновляет счёт на основе пройденной дистанции.

3. Столкновения и завершение
   - При столкновении с препятствием PlayerController уменьшает здоровье (10 для серых сфер и 20 для красных стен), обновляет UI здоровья и уничтожает препятствие.
   - При достижении 0 здоровья вызывается GameManager.EndGame(): показывается LosePanel и игра ставится на паузу.
   - Кнопка на LosePanel вызывает RestartLevel(), загружая сцену 0.

Дополнительные функции:
1. Постепенное ускорение игры: скорость бега увеличивается со временем (корутина SpeedIncrease()), до максимального значения.
2. Поддержка управления свайпами и клавиатурой: свайпы/тап (мышь/тач) + A/D/Space для тестирования в редакторе.
3. Подсчёт очков: UI-счётчик зависит от пройденной дистанции (позиции игрока по Z).
4. Экран поражения с паузой: при проигрыше показывается панель и останавливается время, с возможностью рестарта.
5. Бонусы: реализованы два типа бонусов (сфера зеленого цвета - пополнение здоровья на 10, сфера синего цвета - щит на 5 секунд) с вероятностью генерации 0.05 каждую секунду.
